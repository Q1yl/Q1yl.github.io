一、任务 1：找到黑客植入的 Shell 并获取密码（FLAG：rebeyond）

核心目标

定位黑客上传的 Web 后门（Shell），读取其连接密码。Web 后门（如冰蝎 Shell）是黑客控制服务器的常用工具，通常需密码验证才能连接。

步骤 1：通过 RDP 连接目标服务器

  1. 操作步骤

    ：

    • 在本地 PC 打开「远程桌面连接」（快捷键：Win+R，输入mstsc回车）。
    • 输入目标服务器的 IP 地址（如192.168.1.100，靶场会提供）、登录账号（如管理员账号Administrator）和密码，点击 “连接”。

  2. 操作理由

    ：

    • RDP（远程桌面协议）是 Windows 系统默认的远程管理工具，只有先连接服务器，才能查看其文件和日志。

  3. 原理解析

    ：

    • RDP 通过 TCP 端口 3389 传输数据，允许本地 PC 直接操控服务器桌面，就像操作自己的电脑一样，是运维和溯源的基础入口。

步骤 2：进入 phpstudy 目录，找到 access.log 日志文件

  1. 操作步骤

    ：

    • 服务器桌面通常有phpstudy图标（或在安装目录，默认路径：D:\phpstudy或C:\phpstudy）。
    • 打开 phpstudy 文件夹，依次进入Apache（或 Nginx，取决于 Web 服务器）→ logs目录，找到access.log文件（Web 访问日志，记录所有访问服务器的请求）。

  2. 操作理由

    ：

    • phpstudy是常用的 PHP 环境集成工具，黑客若上传 PHP 类型的 Shell（如冰蝎 Shell），必然会通过 Web 访问该 Shell 文件，访问记录会保存在access.log中。

  3. 原理解析

    ：

    • Web 服务器（如 Apache）会将每一次 HTTP 请求（包括访问的 URL、客户端 IP、访问时间等）写入access.log，这是溯源 “黑客访问路径” 的关键日志。

步骤 3：分析 access.log，定位黑客的 Shell 路径

  1. 操作步骤

    ：

    • 用记事本或 Notepad++ 打开access.log，按Ctrl+F搜索关键词（如shell.php、backdoor.php，黑客常用的 Shell 文件名）。

    • 找到类似如下的日志记录（示例）：

      plaintext

      • 其中/upload/shell.php就是黑客访问的 Shell 路径，对应的文件在服务器的phpstudy网站根目录（默认：D:\phpstudy\WWW）下，即D:\phpstudy\WWW\upload\shell.php。

  2. 操作理由

    ：

    • 黑客要使用 Shell，必须通过 HTTP 请求（如 POST）访问 Shell 文件，日志中的POST方法 + 不常见的 PHP 文件路径，大概率是黑客的 Shell。

  3. 原理解析

    ：

    • 日志格式说明：客户端IP - 用户名 [访问时间] "请求方法 访问路径 协议版本" 状态码 字节数。
    • 状态码200表示请求成功，说明黑客成功访问了该 Shell 文件，路径/upload/shell.php就是 Shell 的位置。

步骤 4：打开 Shell 文件，获取默认密码 “rebeyond”

  1. 操作步骤

    ：

    • 按日志中的路径找到shell.php文件（如D:\phpstudy\WWW\upload\shell.php），用记事本打开。
    • 搜索关键词（如$password、rebeyond），通常冰蝎默认 Shell 的密码会明文或加密存储在代码中，此处直接找到密码为rebeyond。

  2. 操作理由

    ：

    • 冰蝎（Behinder）是主流的 Web 后门工具，其默认生成的 Shell 密码为rebeyond，黑客若未修改默认密码，直接读取即可获取。

  3. 原理解析

    ：

    • 冰蝎 Shell 的核心是 “密码验证 + 命令执行”，代码中会定义一个密码变量（如$pass = 'rebeyond';），只有客户端输入正确密码，才能建立连接并执行黑客的命令。

任务 1 结果

FLAG 为flag{rebeyond}

二、任务 2：分析黑客攻击成功的 IP（FLAG：192.168.126.1）

核心目标

从 Web 访问日志中提取黑客的客户端 IP（即发起攻击的 IP）。

步骤 1：复用任务 1 的 access.log 日志

  1. 操作步骤

    ：

    • 回到任务 1 中打开的access.log文件，找到访问 Shell 文件（如shell.php）的日志记录（示例）：

    • 日志中第一个 IP 地址（192.168.126.1）就是黑客的客户端 IP。

  2. 操作理由

    ：

    • access.log中 “请求来源 IP” 字段（日志最开头）记录了发起访问的设备 IP，黑客通过该 IP 访问 Shell，因此该 IP 就是攻击 IP。

  3. 原理解析

    ：

    • 所有 HTTP 请求都会携带客户端的 IP 地址（除非用代理隐藏），Web 服务器会将该 IP 记录在日志中，这是定位攻击源的最直接证据。

任务 2 结果

FLAG 为flag{192.168.126.1}

三、任务 3：找到黑客创建的隐藏账户（FLAG：hacker138）

核心目标

排查服务器中黑客创建的隐藏账户（通常带$符号或特殊权限，用于长期控制服务器）。

步骤 1：通过 “控制面板” 查看用户账户

  1. 操作步骤

    ：

    • 在服务器桌面，右键点击「此电脑」→ 「管理」（或直接打开「控制面板」→ 「用户账户」→ 「管理账户」）。
    • 进入「计算机管理」窗口，左侧展开「本地用户和组」→ 「用户」，查看所有用户账户。
    • 找到名称带$（Windows 默认隐藏账户标识）或名称异常的账户（如hacker138，非系统默认账户）。

  2. 操作理由

    ：

    • 黑客入侵后，常创建隐藏账户（添加$符号可让账户在 “控制面板 - 用户账户” 中默认不显示），用于后续远程登录（如通过 RDP 或远程命令）。

  3. 原理解析

    ：

    • Windows 系统中，账户名称后加$符号会被标记为 “隐藏账户”，普通用户在桌面看不到，但通过「计算机管理」可强制显示；黑客创建此类账户是为了躲避常规检查，长期驻留服务器。

步骤 2：确认隐藏账户身份

  1. 操作步骤

    ：

    • 右键点击异常账户（如hacker138）→ 「属性」，查看是否有 “管理员权限”（如加入Administrators组），若有则大概率是黑客账户。

  2. 操作理由

    ：

    • 黑客创建的隐藏账户通常会赋予自己管理员权限，以便修改服务器配置、上传文件等，因此 “异常名称 + 管理员权限” 是判断黑客账户的核心依据。

任务 3 结果

FLAG 为flag{hacker138}

四、任务 4：分析黑客挖矿程序的矿池域名（FLAG：wakuang.zhigongshanfang.top）

核心目标

定位黑客上传的挖矿程序，反编译后提取其连接的矿池域名（挖矿程序需连接矿池才能提交算力，获取收益）。

步骤 1：找到黑客账户下的挖矿程序（Kuang.exe）

  1. 操作步骤

    ：

    • 进入黑客隐藏账户（hacker138）的桌面目录（默认路径：C:\Users\hacker138\Desktop）。
    • 找到名称异常的可执行文件（如Kuang.exe，“矿” 的拼音缩写，大概率是挖矿程序）。

  2. 操作理由

    ：

    • 黑客通常会将挖矿程序放在自己创建的账户目录下（避免被管理员发现），且文件名常含 “矿”“mine” 等关键词，方便自己管理。

  3. 原理解析

    ：

    • 挖矿程序是消耗服务器 CPU/GPU 算力的恶意程序，黑客需将其放在可执行目录（如桌面），并通过账户权限运行，因此在黑客账户桌面找挖矿程序是最高效的方式。

步骤 2：用 pyinstxtractor.py 解包 Python 打包的 Kuang.exe

  1. 背景说明：

    • Kuang.exe是用PyInstaller（Python 打包工具）将 Python 脚本打包成的可执行文件，直接打开无法查看源码，需先 “解包” 提取出内部的 Python 字节码文件（.pyc）。
    • pyinstxtractor.py是专门用于解包PyInstaller打包文件的 Python 脚本，需提前在服务器上准备（可从网上下载后复制到服务器）。

  2. 操作步骤：

    • 在服务器上安装 Python 环境（若未安装，从 Python 官网下载并安装，勾选 “Add Python to PATH”）。

    • 将pyinstxtractor.py和Kuang.exe放在同一文件夹（如D:\mine_analysis）。

    • 打开 “命令提示符”（Win+R，输入

      回车），切换到该文件夹：

      cmd

    • 执行解包命令：

      cmd

    • 解包完成后，会生成一个Kuang.exe_extracted文件夹，进入该文件夹，找到Kuang.pyc文件（Python 字节码文件，对应原挖矿脚本）。

  3. 操作理由：

    • PyInstaller打包的exe文件本质是 “Python 解释器 + 脚本字节码 + 依赖库” 的压缩包，pyinstxtractor.py可拆解该压缩包，提取出核心的.pyc字节码文件，为后续反编译做准备。

  4. 原理解析：

    • .pyc文件是 Python 脚本编译后的字节码（非机器码），无法直接阅读，但可通过反编译工具还原为近似原代码的 Python 脚本，从而获取挖矿程序的配置（如矿池地址）。

  4. 原理解析：

    • 矿池是挖矿的 “算力集中管理平台”，所有挖矿设备需连接矿池才能参与挖矿，黑客的挖矿程序会在代码中硬编码矿池地址（避免被轻易修改），反编译后即可直接读取。
