命令拆解（逐段解释作用与原理）

|命令片段|作用|原理说明| |-|-|-| |cut -d- -f 1 access.log.1|提取日志中的所有访问 IP|- cut：按指定分隔符切割文本的工具；- -d-：指定分隔符为 -（Apache 日志中 IP 后紧跟 -，如 192.168.200.2 - - [时间]）； -f 1：保留切割后的第 1 段内容（即 IP 地址）；- 最终输出：所有访问 IP 的列表（含重复）。| |uniq -c|对 IP 去重，并统计每个 IP 的重复次数|- uniq：默认合并 “相邻的重复行”（需配合后续排序确保重复 IP 相邻）；- -c：在每行前添加 “重复次数”（如 6555 192.168.200.2 表示该 IP 出现 6555 次）。| |sort -rn|按 IP 的访问次数 “从大到小” 排序|- sort：文本排序工具；-n：按 “数字” 排序（避免将 100 排在 20 前面）；-r：反向排序（让访问次数最多的 IP 排在最顶部）。| |head -20|只显示排序后的前 20 条结果|- head：默认显示文件前 10 行，-20 表示显示前 20 行；- 作用：无需查看所有 IP，前 20 行足以定位访问次数最多的 IP。|

问题 3：统计 index.php 页面被访问的次数

操作目的

找出 access.log.1 中所有访问 /index.php 页面的请求，统计总次数（靶场结果为 27 次）。

执行命令

bash

命令拆解

|命令片段|作用|原理说明| |-|-|-| |grep "/index.php" access.log.1|过滤出 “包含 /index.php” 的所有日志行|- grep：匹配包含目标页面路径的行；- 原理：Apache 日志中 “访问的页面路径” 包含在第 5 列（如 "GET /index.php HTTP/1.1"），直接匹配 /index.php 即可精准定位所有访问该页面的请求。| |wc -l|统计过滤后的日志行数（即 index.php 的访问次数）|- wc：文本统计工具，-l 是核心参数，表示 “只统计行数”；- 原理：每一行日志对应一次访问请求，行数 = 访问次数。|

问题 4：统计 192.168.200.2 的访问次数（避免误匹配）

操作目的

精准统计目标 IP 192.168.200.2 的总访问次数，避免误匹配类似 192.168.200.211 的 IP。

执行命令

bash

命令拆解

|命令片段|作用|原理说明| |-|-|-| |grep -w "192.168.200.2" access.log.1|精准匹配 “单独作为一个单词” 的目标 IP，排除类似 192.168.200.211 的 IP|- grep -w：-w 是 “全词匹配” 参数，只匹配 “以空白符或标点分隔的独立单词”；- 关键作用：日志中 IP 后紧跟 -（如 192.168.200.2 - - [时间]），IP 是独立单词，不会匹配 192.168.200.211（该 IP 中 2 后无分隔符，不是独立单词）。| |wc -l|统计目标 IP 对应的日志行数（即访问次数）|- 原理同问题 3，行数 = 访问次数，此处统计精准过滤后的 IP 行数。|

问题 5：统计 2023 年 8 月 03 日 8 时内的访问 IP 数量

操作目的

找出 “2023 年 8 月 03 日 8:00-8:59” 这 1 小时内的所有访问 IP，去重后统计 IP 总数。

执行命令

bash

命令拆解

|命令片段|作用|原理说明| |-|-|-| |cat access.log.1|读取日志全部内容|- 原理同问题 2，输出所有日志行，为后续过滤做准备。| |grep "03/Aug/2023:08:"|过滤出 “2023 年 8 月 03 日 8 时”（8:00-8:59）的所有日志行|- 原理：Apache 日志时间格式为 [03/Aug/2023:08:10:22 +0800]，03/Aug/2023:08: 匹配 “8 点开头的所有时间”，精准锁定 1 小时内的请求。| |awk '{print $1}'|提取过滤后日志中的 “访问 IP”（日志第 1 列）|- awk：强大的文本处理工具，{print $1} 表示 “打印每行的第 1 个字段”；- 优势：比 cut 更灵活，直接按默认空格分隔提取 IP，无需指定分隔符。| |sort -nr|对提取的 IP 按 “数字降序” 排序|- sort -n：按数字大小排序（IP 是数字组合，需按数字规则排序，避免字母排序混乱）；- -r：反向排序（从大到小），仅为结果更易读，不影响 IP 去重。| |uniq -c|对排序后的 IP 去重，并统计每个 IP 的访问次数|- uniq：需配合 sort 使用（仅合并相邻重复行），-c 显示重复次数；- 关键作用：去重后 “行数” 即为 “不同 IP 的数量”，次数可辅助判断 IP 活跃度。|
