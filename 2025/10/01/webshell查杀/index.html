webshell查杀

前置知识：核心概念解释

  • Webshell：黑客上传到服务器的恶意脚本文件（如.php/.jsp/.asp），可通过浏览器或工具远程控制服务器（执行命令、窃取数据等），是网站被入侵的核心证据。
  • 免杀 Webshell：经过特殊处理（如代码加密、混淆、伪装），能绕过杀毒软件 / 安全扫描工具检测的 Webshell，更难被发现。
  • D 盾：一款专业的 Webshell 查杀工具，通过 “特征码匹配”“行为分析” 检测恶意脚本，是应急响应中常用的工具。
  • MD5：一种哈希算法，可将任意长度的文本 / 文件转换为 32 位（或 16 位）的固定字符串，常用于 “验证文件唯一性”（同一文件 MD5 固定，修改后会变化）。
  • 隐藏文件：Linux 系统中，文件名以 “.” 开头的文件默认隐藏（用ls命令看不到，需用ls -a查看），黑客常用来隐藏 Webshell 躲避排查。

一、任务 1：获取 Webshell 中的 flag（flag {xxxx-xxxx-xxxx-xxxx-xxxx}）

步骤 1：备份服务器文件（tar -czvf html.tar.gz ./）

  • 原因：靶机有时间限制（“开久了会扣金币”），且直接在服务器操作可能误删文件或被黑客预留的 “自毁程序” 破坏证据，需先保存原始文件到本地分析。
  • 作用：完整保留服务器中的所有文件（包括 Webshell），确保后续分析有完整的证据链，且不依赖靶机运行时间。
  • 操作方法：
  1. 登录靶机（账号 root，密码 xjwebshell），打开终端；
  2. 输入命令 tar -czvf html.tar.gz ./ （“./” 表示当前目录，若 Web 文件在特定目录如/var/www/html，则替换为该路径）；
  3. 等待命令执行完成，会生成一个名为html.tar.gz的压缩包（包含所有文件）。

步骤 2：用 D 盾扫描备份文件，定位可疑 Webshell

  • 原因：人工排查大量文件效率极低，且难以识别伪装的恶意代码（如加密、混淆的 Webshell），需用专业工具快速定位风险文件。

    D 盾通过 “特征码匹配”（识别已知 Webshell 的代码片段）和 “行为分析”（检测危险函数如eval、exec），自动标记高风险文件，大幅缩小排查范围。

  • 操作方法：

  1. 点击 “浏览”，选择解压后的html目录（即从html.tar.gz解压出的文件夹）；(此题为web查杀,所以我们选择扫描www目录下的html文件夹,如果用的连接工具,可直接下载文件到主机)

  2. 点击 “开始扫描”，等待扫描完成，查看结果列表（重点关注 “级别 4” 的文件，风险最高）。

步骤 3：查看可疑文件内容，寻找 flag

  • 原因：黑客常在 Webshell 中用注释（如//flag{...}）隐藏关键信息（方便自己记录），这些注释不会影响代码运行，却可能成为解题线索。
  • 操作方法：

  1. 在 D 盾扫描结果中一个个看，然后找到级别 4 的\html\include\gz.php；

  2. 用文本编辑器（如 Notepad++、VS Code）打开该文件；

  3. 从头到尾浏览代码，重点看注释部分（PHP 中//或/* */包裹的内容）

    发现//027ccd04-5065-48b6-a32d-77c704a5e26d，即为 flag。

二、任务 2：获取黑客工具的 GitHub 地址 MD5（flag {md5}）

步骤 1：提取 Webshell 的特征代码

  • 原因：不同黑客工具（如哥斯拉、蚁剑）生成的 Webshell 有独特的 “代码特征”（如加密函数名、变量名、逻辑结构），这些特征是识别工具的 “指纹”。
  • 操作方法：

  1. 打开 D 盾标记的高风险文件（如gz.php或.Mysqli.php）；

  2. 复制一段核心代码（避免太简单的语句，选包含工具特有函数的片段），例如：

    php

步骤 2：用 GitHub 搜索特征代码，定位工具

  • 原因：黑客工具（即使是恶意工具）常被上传到 GitHub（代码托管平台），通过特征代码搜索可找到工具的官方仓库。
  • 操作方法：
  1. 打开 GitHub 官网（https://github.com），在搜索框粘贴复制的特征代码；
  2. 点击搜索框右侧的 “Code” 标签（只搜索代码内容，排除文档、讨论等无关结果）； ![屏幕截图 2025-09-29 150428](https://raw.githubusercontent.com/3530268799-design/yanqo/main/屏幕截图 2025-09-29 150428.png)
  3. 查看搜索结果，找到匹配的仓库（本题中匹配 “哥斯拉” 工具，地址为https://github.com/BeichenDream/Godzilla）。

步骤 3：计算 GitHub 地址的 MD5 值

  • 原因：题目要求 “GitHub 地址的 MD5”，MD5 可将任意字符串转换为固定 32 位的哈希值，用于唯一标识该地址（避免直接填写地址，符合 flag 格式要求）。
  • 操作方法：

  1. 打开在线 MD5 计算器（如https://md5jiami.51240.com/）；

  2. 在输入框中粘贴 GitHub 地址 https://github.com/BeichenDream/Godzilla；

  3. 点击 “开始加密”，得到 MD5 值39392de3218c333f794befef07ac9257，即为 flag。

    ![屏幕截图 2025-09-29 150757](https://raw.githubusercontent.com/3530268799-design/yanqo/main/屏幕截图 2025-09-29 150757.png)

三、任务 3：获取隐藏 Webshell 的完整路径 MD5（flag {md5}）

步骤 1：识别隐藏的 Webshell

  • 原因：黑客为躲避排查，常将 Webshell 命名为 “隐藏文件”（Linux 中文件名以 “.” 开头的文件默认隐藏，ls命令看不到，需ls -a才显示）。
  • 操作方法：

  1. 查看 D 盾扫描结果，注意文件名以 “.” 开头的文件（如.Mysqli.php）；

  2. 结合风险级别（本题中只有.Mysqli.php为”.”开头，确认是隐藏Webshell），判断为隐藏的恶意文件。

步骤 2：确定隐藏 Webshell 的完整路径

  • 原因：题目要求 “完整路径的 MD5”，需明确文件在服务器中的绝对路径（从根目录/开始的全路径），否则 MD5 计算会出错。
  • 操作方法：
  1. 分析 D 盾显示的路径：\html\include\Db\.Mysqli.php（Windows 格式，“\” 分隔）；
  2. 转换为 Linux 绝对路径：网站文件通常存放在/var/www/目录下，因此完整路径为 /var/www/html/include/Db/.Mysqli.php（Linux 用 “/” 分隔层级）。

步骤 3：计算完整路径的 MD5 值

  1. 打开在线 MD5 计算器，输入完整路径 /var/www/html/include/Db/.Mysqli.php；

  2. 计算得到 MD5 值aebac0e58cd6c5fad1695ee4d1ac1919，即为 flag。

四、任务 4：获取免杀 Webshell 的完整路径 MD5（flag {md5}）

步骤 1：明确 “免杀 Webshell” 的判断标准

  • 原因：免杀 Webshell 是经过特殊处理（如加密、混淆）的恶意文件，能绕过杀毒软件检测，比普通 Webshell 更隐蔽，需通过杀毒软件验证。
  • 作用：区分普通 Webshell 和免杀 Webshell，找到黑客更难被发现的恶意文件。
  • 判断标准：用主流杀毒软件（如火绒、360）扫描所有可疑文件，未被检测为恶意的文件即为免杀 Webshell。

步骤 2：用杀毒软件扫描可疑文件

  • 原因：仅靠 D 盾无法判断是否 “免杀”（D 盾和杀毒软件的检测规则不同），需用杀毒软件实际检测验证。
  • 作用：筛选出能绕过杀毒软件的免杀 Webshell。
  • 操作方法：
  1. 下载并安装杀毒软件（如火绒安全）；
  2. 将 D 盾扫出的 4 个可疑文件（shell.php、gz.php、.Mysqli.php、top.php）拖入杀毒软件进行扫描；
  3. 观察结果：本题中前 3 个被检测为恶意，top.php未被检测，因此top.php是免杀 Webshell。

步骤 3：确定免杀 Webshell 的完整路径并计算 MD5

操作方法：

  1. 确定top.php的完整路径：D 盾显示路径为\html\wap\top.php，转换为 Linux 绝对路径为 /var/www/html/wap/top.php；
  2. 用在线 MD5 计算器计算该路径，得到eeff2eabfd9b7a6d26fc1a53d3f7d1de，即为 flag。
