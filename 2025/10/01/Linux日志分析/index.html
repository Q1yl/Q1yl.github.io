1.有多少 IP 在爆破主机 SSH 的 root 帐号

1./var/log是 Linux 默认日志目录，比如 SSH 登录日志、系统操作日志都存在这里，是分析的核心素材。

2.输入命令tar -czvf log.tar.gz ./

代码解释

  • tar -czvf是 “打包 + 压缩” 命令：-c创建压缩包，-z用 gzip 压缩（让文件更小），-v显示压缩过程，-f指定压缩包名（这里是log.tar.gz），./表示 “当前目录下所有文件”。

3.**auth.log.1文件:**这是 SSH 登录日志的 “历史文件”，因为 Linux 会定期滚动日志，旧日志会加.1后缀

(SSH 登录（包括爆破、成功登录）的所有记录，都会存在auth.log（新日志）或auth.log.1（旧日志）中)

4.“爆破 root 账号” 的特征是 “多次尝试用 root 账号登录，但密码错误”，日志中会显示Failed password for root（“root 账号密码错误”），只要提取这些日志中的 IP，去重后就是答案。

命令:cat auth.log.1 | grep -a "Failed password for root"

命令解释

  • cat auth.log.1：把auth.log.1日志文件的内容 “读出来”（cat是 “查看文件内容” 命令）。
  • |：管道符，相当于 “把前一个命令的结果传给后一个命令处理”（比如把日志内容传给grep过滤）。
  • grep -a "Failed password for root"：grep是 “搜索过滤” 命令，-a确保能正常读取日志（避免日志编码问题），"Failed password for root"是 “搜索关键词”—— 只保留包含 “root 账号密码错误” 的日志行。

5.若IP很多需要去重

输入命令：cat auth.log.1 | grep -a "Failed password for root" | awk '{print $11}' | uniq -c

命令解释

  • awk '{print $11}'：awk是 “提取列” 工具，日志中 IP 在第 11 列（比如第一行日志，192.168.200.32是第 11 个元素），所以用print $11提取 IP。

  • uniq -c：uniq是 “去重” 命令，-c表示 “统计每个 IP 出现的次数”（即每个 IP 爆破 root 的次数）。

    所以flag是flag{192.168.200.2,192.168.200.31,192.168.200.32}

2.SSH 爆破成功登陆的 IP 是多少

1.“SSH 登录成功” 的日志特征是Accepted password for root（“root 账号密码正确，登录成功”）

2.提取登录成功的日志

代码:cat auth.log.1 | grep -a "Accepted "

我们看到只有一个IP登录成功

所以flag就是flag{192.168.200.2}

3.爆破用户名字典是什么

核心逻辑：

“爆破用户名字典” 是黑客尝试登录的所有账号（包括存在的账号如 root，和不存在的账号如 test1），日志中 “密码错误” 的记录会显示Failed password for 用户名（比如Failed password for invalid user test1表示 “尝试用不存在的账号 test1 登录，密码错误”），提取这些 “用户名” 去重后就是字典。

1.提取所有 “密码错误” 的日志

输入命令：cat auth.log.1 | grep -a "Failed password"

输出结果中会有以下内容：

注释:invalid user翻译为不存在的用户invalid user后面的名字就是尝试的用户名

2.提取 “用户名”

输入命令：cat auth.log.1 | grep -a "Failed password" | grep -o 'for .* from' | uniq -c

  • 命令解释：

    • grep -o 'for .* from'：-o表示 “只提取匹配的部分”，for .* from是 “从 for 开始，到 from 结束的所有内容”（比如for root from、for invalid user user from），这样就能定位到 “用户名” 所在的片段。

    • uniq -c：去重并统计每个用户名被尝试的次数，避免重复的用户名干扰结果。

      由此可知flag就是flag{user,hello,root,test3,test2,test1}

4.登陆成功的 IP 共爆破了多少次

核心逻辑：

“登录成功的 IP” 是问题 2 中找到的192.168.200.2，“爆破次数” 是这个 IP 尝试登录root账号的总失败次数（因为爆破是 “多次尝试”，直到成功，所以失败次数就是爆破次数）。

提取 “192.168.200.2 尝试登录 root” 的所有日志

输入命令：cat auth.log.1 | grep -a "192.168.200.2" | grep "for root"

  • 命令解释：

    • grep -a "192.168.200.2"：先过滤出这个 IP 的所有日志。

    • 再用grep "for root"：过滤出这个 IP 尝试登录root账号的日志（因为爆破的是 root 账号）。

      从图片中可以看到,登录失败(Failed password )总共有4次

      所以flag就是flag{4}

5.黑客登陆主机后新建了一个后门用户，用户名是多少

核心逻辑：

“新建用户” 在 Linux 中会通过useradd命令执行，日志中会显示new user: name=用户名（“新建用户，名字是 XXX”）；同时，Linux 所有用户信息都存在/etc/passwd文件中，也能直接查看。

详细步骤：

方法1:

从日志中找 “新建用户” 记录

输入命令：cat auth.log.1 | grep -a "new user"，按回车。

  • 命令解释：new user是 “新建用户” 的日志关键词，直接过滤就能找到黑客新建的用户（name=test2）。

    可以看到有两条new user,查阅资料可知

    日志中若有类似 uid=1001、home=/home/debian、shell=/bin/bash，则基本可以确定是普通用户。

    所以flag就是flag{test2}
